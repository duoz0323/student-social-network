# Workflow review code

## 1. Phạm vi review

Kiểm tra:

- Đúng nghiệp vụ.
- Đúng kiến trúc.
- Bảo mật.
- Hiệu năng.
- Validation.
- Phân quyền.
- Khả năng bảo trì.
- Test.
- Comment.
- Tài liệu.

## 2. Mức độ vấn đề

- Critical: lỗ hổng, mất dữ liệu, sai quyền nghiêm trọng.
- High: lỗi nghiệp vụ chính hoặc crash.
- Medium: hiệu năng, maintainability hoặc edge case.
- Low: naming, style hoặc cải thiện nhỏ.

## 3. Cách báo cáo

Mỗi vấn đề gồm:

- Mức độ.
- File/dòng.
- Mô tả.
- Nguyên nhân.
- Ảnh hưởng.
- Hướng sửa.

Không chỉ đưa nhận xét chung chung.
